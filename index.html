Примеры сценариев

Разобраться в чужой таблице

Таблица с несколькими листами, формулами, непонятными связями.

"Объясни, как устроена эта таблица: какие листы, откуда берутся данные, как считаются итоги"

Агент читает все листы, находит зависимости, объясняет логику.

Вывод формулы из описания задачи

Пользователь знает что хочет посчитать, но не знает какой функцией.

"Посчитай среднюю выручку без учёта нулевых" → =AVERAGEIF(...,"<>0")

Агент пишет формулу, объясняет как она работает, записывает в ячейку.

Аудит и исправление ошибок в формулах

Таблица используется давно, редактировалась разными людьми.

"Проверь формулы — нет ли ошибок?"

Агент находит битые ссылки, несогласованные диапазоны, отличия от паттерна столбца.

Создание шаблона таблицы с формулами

Нужна таблица определённой структуры для регулярного использования.

"Сделай шаблон учёта командировочных с формулами итогов"

Вход: описание. Выход: .ods с формулами и форматированием.

Аналитическая записка по данным из таблицы

Есть таблица с цифрами — нужен документ для руководства.

"Напиши записку: ключевые показатели, тренды, на что обратить внимание"

Вход: .ods. Выход: .odt с таблицами и выводами.

Перевод документа с сохранением структуры

"Переведи на русский, сохрани форматирование"

Вход: .odt на английском. Выход: .odt на русском, стили сохранены.

Применение в российском контексте

Аналитик в бюджетном учреждении → аналитическая записка

Кассовое исполнение бюджета по 12 подразделениям за год. Нужно подготовить пояснительную записку для ГРБС — где перерасход, где экономия, какие статьи требуют корректировки.

"На листе 'Факт' — кассовое исполнение, на листе 'План' — бюджетные назначения. Сравни по КВР, покажи где отклонение больше 10%, напиши записку с выводами"

Агент: читает оба листа, находит расхождения по статьям, генерирует .odt с таблицей отклонений и текстовым анализом.

Финансист → создание шаблона с формулами

Ежемесячный отчёт по исполнению госконтрактов (ФЗ-44): одна структура, но данные обновляются каждый период.

"Сделай шаблон учёта контрактов: номер, предмет, НМЦК, цена контракта, оплачено, остаток, % исполнения, срок, статус. Формулы для вычисляемых столбцов, итоги по группам"

Агент: создаёт .ods с формулами, готовый к ежемесячному заполнению.

Новый сотрудник → разобраться в чужой таблице

После миграции с MS Office на отечественное ПО получил таблицу, которую вёл предшественник 3 года. Часть формул сломалась при конвертации.

"Объясни структуру файла. Какие листы справочные, какие расчётные? Есть ли ошибки в формулах после конвертации из .xlsx?"

Агент: читает все листы, строит карту зависимостей, находит битые формулы, объясняет структуру.

Суверенитет данных — ключевое преимущество

  • Данные не уходят за рубеж (Microsoft Copilot / Google Gemini — облако US)
  • Open-source модели (Qwen 3.5 397B, уровень GPT-4o/Claude) можно развернуть на собственных серверах
  • Российские LLM (YandexGPT, GigaChat) поддерживают вызов инструментов, но пока уступают по качеству — требуется тестирование
  • Соответствие ФЗ-152 о персональных данных

Наш прототип

Работающий прототип ИИ-агента для офисного пакета. Демонстрирует, что технология работает.

Что реализовано

  • Создание таблиц (.ods) по описанию — с формулами, форматированием, итогами
  • Создание документов (.odt) — с заголовками, таблицами, списками
  • Чтение и анализ существующих файлов — агент отвечает на вопросы по данным
  • Перевод документов с сохранением структуры
  • Точечное редактирование — агент может изменить конкретную ячейку или формулу

Модель

Qwen 3.5 397B (Alibaba, open-source) — одна из сильнейших для работы с инструментами. Не зависит от западных облаков, доступна для развёртывания на собственных серверах.

Разобраться в чужой таблице

Файл с 5 листами, формулы между листами. Пользователь просит объяснить структуру.

Вывод формулы из описания на естественном языке

Пользователь просит добавить столбец «Доля Москвы» в процентах. Агент сам находит нужные столбцы и пишет формулу.

Аналитическая записка по данным из таблицы

Агент читает .ods с продажами по филиалам, анализирует выполнение плана, генерирует .odt с таблицами и выводами.
